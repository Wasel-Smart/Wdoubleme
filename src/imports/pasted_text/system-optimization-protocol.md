SYSTEM COMMAND — WASEL SYSTEM OPTIMIZATION & PERFECTION PROTOCOL

Objective

Perform a full-system audit, optimization, and architectural refinement of the entire platform.

Your mission is to transform the system into a clean, scalable, production-grade platform by:

• Identifying weaknesses
• Removing redundancy
• Eliminating unnecessary components
• Simplifying architecture
• Improving performance
• Strengthening security
• Standardizing development patterns

The final system must achieve elite engineering quality comparable to top global technology platforms.

────────────────────────────────

GLOBAL SYSTEM RULES

1. Analyze the entire codebase and architecture before making changes.

2. Remove any:
   • Duplicate logic
   • Redundant components
   • Unused files
   • Unnecessary dependencies
   • Over-engineered modules
   • Dead code

3. Simplify whenever possible while maintaining functionality.

4. Maintain backward compatibility for critical features.

5. Standardize all patterns across the system.

6. Optimize for scalability, maintainability, and performance.

────────────────────────────────

ARCHITECTURE OPTIMIZATION

Refactor the system into a clear modular architecture.

Principles to enforce:

• Separation of concerns
• Modular design
• Feature-based organization
• Clear service boundaries
• Reusable components

Eliminate tightly coupled logic and circular dependencies.

Ensure that every module has a clear responsibility.

────────────────────────────────

PERFORMANCE OPTIMIZATION

Improve system performance across all layers.

• Faster loading times

• Efficient database queries
• Optimized API responses
• Smart caching strategies
• Lazy loading where appropriate

Target a fast, responsive experience on all devices.
