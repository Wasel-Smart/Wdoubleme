**COMMAND FOR FIGMA AGENT — MOBILITY OS LIVE STREAM MAP REDESIGN**

**Objective:**
Redesign the **Live Stream Map** in Mobility OS to be **ultra-clean, highly readable, techy, digital, and data-driven**. The map must be **centered, visually striking, intuitive, and fully optimized for real-time city-to-city mobility tracking**. All icons, shapes, colors, and flow indicators must be redesigned using **best UX/UI practices**.

---

# 1️⃣ Map Layout & Positioning

* **Map Position:** Fully centered in the viewport
* **Surrounding Space:** Minimal side panels; focus is on map flow
* **Interactive Zones:** Hover, click, zoom, pan clearly defined
* **Background:** Dark-light balanced gradient to enhance routes (optional subtle tech grid overlay)

---

# 2️⃣ Color & Theme Guidelines

**Primary Color Scheme:** Unified, techy, digital:

| Element             | Color                              |
| ------------------- | ---------------------------------- |
| Ride Routes         | #0066FF (Deep Digital Blue)        |
| Package Routes      | #00C2FF (Cyan)                     |
| Congestion / Alerts | #FF3B30 (Red)                      |
| Medium Traffic      | #FF9500 (Orange)                   |
| Low Traffic         | #34C759 (Green)                    |
| Map Background      | #F5F7FA (Clean Light)              |
| Grid / Overlay      | #E6ECF5 (Subtle)                   |
| Text / Labels       | #0A2540 Primary, #5B6B7A Secondary |

**Rules:**

* Colors must feel **unified and harmonious**
* High-contrast for active elements, subtle for background layers
* Avoid visual clutter

---

# 3️⃣ Map Components

**Cities / Nodes:**

* Shape: Circles, hexagons, or tech-inspired polygon hubs
* Glow/Highlight for major hubs
* Hover: show city stats (active rides, packages, traffic)
* Size: scalable by importance

**Routes / Lines:**

* Solid for rides, dashed for packages
* Thickness proportional to traffic load
* Animated particle flow to simulate real-time movement
* Optional: subtle gradient along the line for flow direction

**Icons:**

* Replace all existing icons with **modern minimal, tech-inspired variants**
* Vehicles = moving dots or abstract digital particles
* Packages = small flowing capsules or minimal icons
* Grid, checkpoints, and control points = simplified geometric shapes

---

# 4️⃣ Flow Key / Legend

* Replace old key with **centralized, intuitive flow legend**
* Include:

  * Ride route indicators
  * Package route indicators
  * Traffic density icons (Green/Orange/Red)
  * Active vehicle counts
* Use **techy iconography** rather than text-heavy legends
* Position: Overlay at map corner or integrated into panel

---

# 5️⃣ User Interaction

* Zoom in/out: smooth, techy animation
* Pan: fluid drag
* Hover city/node → shows dynamic tooltip
* Click route → highlight, show detailed metrics
* Optional: toggle layers (rides/packages/traffic)

---

# 6️⃣ Live Data Integration

* Each city/node shows **real-time metrics**:

  * Active rides/vehicles
  * Packages in transit
  * Traffic congestion
  * Estimated arrival times
* Flow animation speed reflects **real-time data**
* Optional subtle pulse effect for high-activity nodes

---

# 7️⃣ Cleanliness & Readability

* Prioritize essential information
* Avoid overlapping lines and clutter
* Use spacing, thickness, and opacity to create hierarchy
* Minimal labels, auto-hide less important details
* Techy digital aesthetic: slight glow, subtle grid, modern font

---

# 8️⃣ Layout & Panel Suggestions

* Map: center, largest visual component
* Optional: collapsible side panel for live metrics
* Minimal top bar: search, filter, toggle layers
* Bottom bar: playback controls (pause, rewind, speed multiplier)

---

# 9️⃣ Icons & Shapes

* Redesign all icons to **flat, minimal, digital style**
* Nodes = scalable geometric shapes
* Vehicles/packages = smooth animated points/particles
* Flow lines = animated gradients to show movement

---
