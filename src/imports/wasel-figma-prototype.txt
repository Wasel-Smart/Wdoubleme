**Brand: Wasel | واصل (One Platform, Multiple Services)**

- Secondary color: #D9965B (warm bronze) — for package delivery feature accent
- Target: Everyone (travelers, passengers, package senders, students, families)
- Services: Carpooling + Package Delivery (integrated features)

**Design System:**
- Background: #0B1120 (deep navy) — dark-first
- Cards: #111B2E with glassmorphism
- Primary CTA: #04ADBF (teal)
- Package feature accent: #D9965B (bronze) — used for package-related buttons/badges
- Success: #10B981 (green)
- Error: #EF4444 (red)
- Typography: Tajawal (Arabic), Inter (English) — bilingual
- Tone: Friendly, culturally-aware, trustworthy, simple
- Currency: JOD (Jordanian Dinar)

**Design Tokens:**
- All colors, spacing, typography must come from `/tokens/wasel-tokens.ts`
- Use CSS variables in `/styles/globals.css`
- NO hardcoded values (enables theme switching)

**Single Brand Benefits:**
- ✅ Simpler marketing (one name to remember)
- ✅ Reduced user confusion (one app, not two)
- ✅ Lower development costs (one codebase)
- ✅ Unified brand experience
- ✅ Easier to explain ("Wasel does rides + packages")

4️⃣ **UI/UX FLOWS (USER JOURNEYS)**

**Journey 1: PASSENGER (Booking a Ride)**
1. Home Screen → "Search Rides" (Wasel brand)
2. Search Form:
   - Origin: Amman (autocomplete)
   - Destination: Aqaba (autocomplete)
   - Date: March 15 (calendar picker)
   - Filters: Gender preference, Prayer stops, Seats needed
3. Search Results (sorted by relevance):
   - Trip Card: Photo, Name, Rating (4.8★), Price (JOD 10), Seats (2/3)
   - Gender badge: "Women-only 🚺" or "Mixed"
   - Prayer stops badge: "2 prayer stops 🕌"
4. Trip Details Modal:
   - Traveler profile (trust score, verified badge)
   - Vehicle info
   - Route map with prayer stops
   - Transparent pricing breakdown
   - "Book Seats" CTA
5. Booking Confirmation:
   - Select seats (1-3)
   - Payment method (card or Cash on Arrival)
   - Trip summary
   - Confirm booking → Success toast
6. My Trips Dashboard:
   - Upcoming trip card (countdown timer)
   - Driver contact (masked until 24h before)
   - Share trip (emergency contact)
7. Post-Trip:
   - Rate traveler (1-5 stars)
   - Write review (optional)
   - Refer friends (earn JOD 2)

**Journey 2: TRAVELER (Posting a Ride + Raje3)**
1. Home Screen → "Post Ride" (Wasel brand)
2. Post Ride Form:
   - Route: Amman → Aqaba
   - Date & Time: March 15, 8:00 AM
   - Seats available: 3
   - Price calculator: Auto-fill JOD 10/seat (based on distance)
   - Gender preference: Mixed / Women-only / Men-only / Family
   - Prayer stops: Toggle ON (auto-calculate)
   - Allow packages: Toggle ON
3. Preview & Publish:
   - Review trip details
   - Publish → Success toast
4. Recommendation Widget (Raje3):
   - 🔥 "8 passengers waiting for Aqaba → Amman (return trip)"
   - "Earn JOD 30 on your return"
   - CTA: "Post Return Trip" (pre-filled form)
5. Traveler Dashboard:
   - Active trip: 3 bookings (passengers' names, photos)
   - Package opportunities: "2 packages waiting (earn JOD 8)"
   - Accept/Decline packages
6. Trip Day:
   - QR scanner (verify passenger pickup)
   - GPS tracking (optional)
   - In-trip chat
7. Post-Trip:
   - Rate passengers (1-5 stars)
   - View earnings: JOD 30 (rides) + JOD 8 (packages) - JOD 4.56 (commission) = JOD 33.44
   - Payout to bank (weekly)

**Journey 3: SENDER (Sending a Package via Awasel)**
1. Home Screen → "Send Package" (Awasel brand)
2. Send Package Form:
   - Route: Amman → Aqaba
   - Weight: 2 kg (slider)
   - Dimensions: 30×20×15 cm
   - Description: "Books"
   - Receiver: Name + Phone
   - Delivery price: JOD 5 (suggested)
   - Insurance: Toggle ON (+JOD 0.50)
3. Browse Available Trips:
   - Show travelers going Amman → Aqaba
   - Trip Card: Date, Time, Traveler rating, "Accept Package" CTA
4. Package Accepted:
   - Notification: "Ahmad accepted your package"
   - QR code generated (pickup)
5. Pickup Day:
   - Traveler scans QR code → Package "in transit"
   - GPS tracking (live location)
6. Delivery:
   - Receiver shows QR code
   - Traveler scans → Package "delivered"
   - Payment released (JOD 4 to traveler, JOD 1 commission)
7. Post-Delivery:
   - Rate traveler (1-5 stars)
   - File insurance claim (if damaged)

9️⃣ **DESIGN SYSTEM (REUSABLE COMPONENTS)**

**Components Library:**

1. **Buttons:**
   - Primary (CTA): #04ADBF, rounded-lg
   - Secondary: outline, hover state
   - Ghost: transparent, text only
   - Icon button: square, icon-only

2. **Cards:**
   - Trip Card: Photo, name, rating, price, seats, badges
   - Package Card: Weight, dimensions, price, status
   - Stat Card: Icon, metric, change percentage
   - Glass Card: Glassmorphism, backdrop-blur

3. **Forms:**
   - Input: Border, focus ring, error state
   - Select: Dropdown, searchable (cities)
   - Date Picker: Calendar view
   - Toggle: Switch for boolean settings
   - Slider: Range input (weight, price)

4. **Navigation:**
   - Sidebar: Collapsible, icon + label
   - Tab Bar: Mobile bottom navigation
   - Breadcrumbs: Page hierarchy

5. **Feedback:**
   - Toast: Success, error, info, warning
   - Modal: Overlay, close button
   - Alert Dialog: Confirm/cancel actions
   - Progress Bar: Loading, percentage
   - Badge: Status, count, feature flag

6. **Data Display:**
   - Table: Sortable, paginated
   - Chart: Bar, line, pie, donut
   - Avatar: User photo, fallback initial
   - Rating: Star display (1-5)
   - Timeline: Trip progress, package tracking

**Naming Convention:**
- Component: PascalCase (TripCard, BookingModal)
- Variants: snake_case (primary, secondary, ghost)
- States: is_active, is_loading, has_error

**Auto-Layout:**
- All frames use Figma auto-layout
- Responsive breakpoints: Mobile (375px), Tablet (768px), Desktop (1440px)
- Grid system: 12 columns, 24px gutter
